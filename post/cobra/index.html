spf13/cobra库学习#

cobra#

cobra是一个命令行程序库，提供简单的接口来创建CLI接口(Command Line Interface)命令行接口。cobra被应用于很多项目比如k8s,docker,etcd,hugo博客等等

命令模板#

  • commands
  • argumets
  • flags

例如:

# server是commands，port是flag
hugo server --port=1313
# clone是commands，URL是arguments，brae是flags
git clone URL --bare

业务使用#

安装#

# go库安装
go get github.com/spf13/cobra/cobra@latest
# go的命令行脚手架安装
go install github.com/spf13/cobra-cli@latest

创建命令#

package cmd

import (
	"fmt"
	"github.com/spf13/cobra"
	"github.com/spf13/cobra/doc"
	"os"
)

var Verbose bool
var Source string
var Region string
var author string
var MarkdownDocs bool
var rootCmd = &cobra.Command{
	Use:   "hugo", // 使用
	Short: "Hugo is a very fast static site generator", // 简单介绍
	Long: `A Fast and Flexible Static Site Generator built with
                love by spf13 and friends in Go.
                Complete documentation is available at https://gohugo.io`,
	//TraverseChildren: true, //默认情况下Cobra仅仅解析目标命令上的本地标志，忽略父命令上的本地标志，通过启用该属性可以解析每个命令的本地标志
	//Args: cobra.MaximumNArgs(2), // 内置的验证函数，位置参数多于2则报错
	Args: func(cmd *cobra.Command, args []string) error {
		// 自定义逻辑校验参数
		return nil
	},
	PersistentPreRun: func(cmd *cobra.Command, args []string) {
		fmt.Println("hugo PersistentPreRun")
	},
	PreRun: func(cmd *cobra.Command, args []string) {
		fmt.Println("hugo PreRun")
	},
	Run: func(cmd *cobra.Command, args []string) {
		fmt.Println("run hugo...")
		fmt.Printf("Verbose: %v\n", Verbose)
		fmt.Printf("Source: %v\n", Source)
		fmt.Printf("Region: %v\n", Region)
	},
	PostRun: func(cmd *cobra.Command, args []string) {
		fmt.Println("hugo PostRun")
	},
	PersistentPostRun: func(cmd *cobra.Command, args []string) {
		fmt.Println("hugo PersistentPostRun")
	},
}

func Execute() {
	if err := rootCmd.Execute(); err != nil {
		fmt.Println(err)
		os.Exit(1)
	}
}
func init() {
	rootCmd.Flags().BoolVarP(&MarkdownDocs, "md-docs", "m", false, "gen Markdown docs")
}

func GenDocs() {
	if MarkdownDocs {
		if err := doc.GenMarkdownTree(rootCmd, "D:\\code\\gocode\\go_zero\\cmd"); err != nil {
			fmt.Println(err)
			os.Exit(1)
		}
	}
}

使用命令标志#

持久标志#

如果该标志是持久的则他适用于该命令下的所有子命令。对于全局标志一般定义在rootCmd

var Verbose bool
rootCmd.PersistentFlags().BoolVarP(&Verbose, "verbose", "v", false, "verbose output")
本地标志#

标志也可以是本地的，这意味着它只适用于该指定命令。

var Source string
rootCmd.Flags().StringVarP(&Source, "source", "s", "", "Source directory to read from")

Hooks#

在执行 Run 函数前后，我么可以执行一些钩子函数，其作用和执行顺序如下：

  1. PersistentPreRun：在 PreRun 函数执行之前执行，对此命令的子命令同样生效。
  2. PreRun：在 Run 函数执行之前执行。
  3. Run：执行命令时调用的函数，用来编写命令的业务逻辑。
  4. PostRun：在 Run 函数执行之后执行。
  5. PersistentPostRun：在 PostRun 函数执行之后执行，对此命令的子命令同样生效。

每个hook方法对应都有一个-E版本的方法，且优先级高于对应的方法。

生成文档#

Cobra 支持生成 Markdown、ReStructured Text、Man Page 三种格式文档。

hugo/cmd/root.go

var MarkdownDocs bool

func init() {
    rootCmd.Flags().BoolVarP(&MarkdownDocs, "md-docs", "m", false, "gen Markdown docs")
    ...
}

func GenDocs() {
    if MarkdownDocs {
        if err := doc.GenMarkdownTree(rootCmd, "./docs/md"); err != nil {
            fmt.Println(err)
            os.Exit(1)
        }
    }
}

在 main.go 中调用 GenDocs() 函数。

func main() {
    cmd.Execute()
    cmd.GenDocs()
}

Cobra命令行使用#

#下载包
go install github.com/spf13/cobra-cli@latest
#帮助命令
cobra-cli -h 
#初始化程序
cobra-cli init

添加命令#

$ cobra-cli add serve
$ cobra-cli add config
$ cobra-cli add create -p 'configCmd' --author "jianghushinian" --license mit --viper

注意⚠️：使用 -p 'configCmd' 标志指定当前命令的父命令时，configCmd 必须是小驼峰命名法，因为 cobra-cli 为 config 生成的命令代码自动命名为 configCmd，而不是 config_cmd 或其他形式，这符合 Go 语言变量命名规范。

viper库#

viper可以完成工作#

  1. 查找，加载和反序列化json toml yaml envfile
  2. 监控配置文件改动，并==热加载==配置文件
  3. 从远程配置中心读取(etcd/consul)
  4. 从命令行标志中获取
  5. 可以轻松区分用户提供的命令行参数或者配置文件中的值是否与默认值相同。

viper优先级#

  • 显式调用 viper.Set 设置的配置值
  • 命令行参数
  • 环境变量
  • 配置文件
  • key/value 存储
  • 默认值

使用#

  • 安装

    go get github.com/spf13/viper

  • 文档

    下载依赖后在Go Modules中找到viper目录，其中有个README.md 指导文档

Pflag库#

参考地址

go 标准库提供了flag包实现命令行参数解析，但其只满足基本需要，不支持高级特性。Pflag就是替代flag的。

特点#

pflag作为flag的包替代品

  • 实现了POSIX/GNU风格的-flags
  • Pflag语法兼容
  • 兼容 Go 标准库中的 flag 包。如果直接使用 flag 包定义的全局 FlagSet 对象 CommandLine，则完全兼容；否则当你手动实例化了 FlagSet 对象，这时就需要为每个标志设置一个简短标志（Shorthand）。
package main

import (
	"fmt"

	"github.com/spf13/pflag"
)

type host struct {
	value string
}

func (h *host) String() string {
	return h.value
}

func (h *host) Set(v string) error {
	h.value = v
	return nil
}

func (h *host) Type() string {
	return "host"
}

func main() {
	var ip *int = pflag.Int("ip", 1234, "help message for ip")

	var port int
	pflag.IntVar(&port, "port", 8080, "help message for port")

	var h host
	pflag.Var(&h, "host", "help message for host")

	// 解析命令行参数
	pflag.Parse()

	fmt.Printf("ip: %d\n", *ip)
	fmt.Printf("port: %d\n", port)
	fmt.Printf("host: %+v\n", h)

	fmt.Printf("NFlag: %v\n", pflag.NFlag()) // 返回已设置的命令行标志个数
	fmt.Printf("NArg: %v\n", pflag.NArg())   // 返回处理完标志后剩余的参数个数
	fmt.Printf("Args: %v\n", pflag.Args())   // 返回处理完标志后剩余的参数列表
	fmt.Printf("Arg(1): %v\n", pflag.Arg(1)) // 返回处理完标志后剩余的参数列表中第 i 项
}

pflag和flag互相兼容，可以混用。
