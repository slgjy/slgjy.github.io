Maven私有仓库#

为什么搭建私有库：

  1. 公共仓库下载慢。
  2. 项目组之间需要共享包。

Nexus#

Sonatype Nexus是一个强大的Maven仓库管理器。官网地址

Nexus当前有Nexus2.x和3.x两大版本，两个版本都在更新迭代，两个版本互不影响都可以使用。

Nexus分为pro和oss版本，其中oss是开源免费的，pro是收费版本。

1.下载包#

本次使用3.x版本,下载地址

2.上传解压#

tar -zxvf  nexus-3.54.1-01-unix.tar.gz

  • nexus用于实现nexus功能
  • sonatype-work： 用于存储数据

3.修改配置文件#

1.修改Nexus端口#

在nexus下的etc目录下的配置文件nexus-default.properties的application-port配置

4.启动#

 # bin目录下执行执行 ./nexux start 命令启动。脚本涵盖：./nexus {console | start | stop | restart | status | dump}
 ./nexus start
 # 3.x的nexus启动
 nohup ./bin/nexus run > logs/nexus.log 2>&1 &

访问：ip+端口

切换非root用户启动#

官方不建议用root用户启动

  1. 新增用户

    adduser nexus

  2. 在 bin/nexus.rc 指定专用用户，配置run_as_user="nexus"

  1. 修改nexus的属组

    chown -R nexus:nexus /opt/nexus /opt/sonatype-work

通过如下修改没有启动成功

删除了之前通过root用于解压的包，然后通过新建用户重新解压启动成功。

Nexus配置#

1. 登录配置#

使用admin用户登录nexus，按提示重置密码，并设置邮箱方便密码找回。

默认账号和密码：admin/admin123（3.X密码在data/admin.password）

2.设置Blob stores#

nexus3使用了Blob Stores来保存仓库文件，穿件仓库时使用，可以自定义存储路径路径。也可使用默认的配置。

3.创建仓库#

​ 阿里云仓库地址：http://maven.aliyun.com/nexus/content/groups/public/

4.添加slg-hosted仓库#

5.添加设置宿主仓库#

添加设置宿主仓库release、snapshots(hosted自身项目用仓库，也可用默认自带的)

6.设置仓库组#

添加设置仓库组maven2（group），仓库组可用默认的maven-public（添加组成员release、snapshot、slg-hostsed、阿里云、中央，因为按顺序遍历，所以远程的放后面，阿里云和华为云类似的要比其他代理的快，放其他远程前面）

7.创建角色roles#

为项目创建项目应有权限的角色，与仓库绑定。添加用户时可绑定角色。

8.创建用户#

Nexus包上传#

1.mavne配置#

本地settings.xml配置

 <!--配置账户信息-->
<server>
  <id>devloper</id>
  <username>slgDevloper</username>
  <password>slg8610..</password>
</server>
 <!--配置仓库地址信息（这里配置的group地址，他关联了其他的库）-->
<mirror>
  <id>slg-test-group</id>
  <mirrorOf>slg-test-group</mirrorOf>
  <name>slg-test-group</name>
  <url>http://xxxx:8300/repository/slg-test-group/</url>
</mirror>

2.idea配置#

pom文件配置

 <distributionManagement>
        <repository>
             <!--和本地settings.xml配置文件server模块的id保持一致，用于鉴权操作-->
            <id>devloper</id>
            <!--需要部署的仓库地址-->
            <url>http://xxxx:8300/repository/slg-test-release/</url>
        </repository>
        <snapshotRepository>
            <id>devloper</id>
            <url>http://xxxx:8300/repository/slg-test-snapshot/</url>
        </snapshotRepository>
    </distributionManagement>
