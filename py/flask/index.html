蓝图#

Blueprint是flask程序的模块化处理机制

当业务越大逻辑越复杂，需要对功能进行模块化分割处理。将不同的业务单元分割成多个文件来实现。是项目结构清洗，易于维护。

  • 一个项目可以具有多个Blueprint
  • Blueprint可以单独拥有模板，静态文件目录

目录结构

machine.py

from flask import Blueprint
# 创建蓝图，url_prefix添加统一的路由前缀
bp = Blueprint('machine', __name__, url_prefix='/machine')
@bp.route('/list')
def machine_list():
    return 'machine---list'
@bp.route('/add')
def machine_add():
    return 'machine---add'

user.py

from flask import Blueprint
bp = Blueprint('user', __name__, url_prefix='/user')
@bp.route('/list')
def user_list():
    return 'user---list'
@bp.route('/add')
def user_add():
    return 'user---add'

app.py

from flask import Flask
from apps.machine import bp as machine_bp
from apps.user import bp as user_bp
app = Flask(__name__)
app.register_blueprint(machine_bp)
app.register_blueprint(user_bp)
if __name__ == '__main__':
    app.run()
